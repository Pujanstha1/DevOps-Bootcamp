# 4. Variables and Parameter Expansion

## Variables
- Variables store data that can be used later in scripts.
- No spaces around `=` when assigning.

### Assigning Variables
```bash
name="Pujan"
age=25
```

### Using Variables
```bash
echo "Name: $name, Age: $age"
```
- `$variable` or `${variable}` expands the variable value

---

## Types of Variables
- **Local variables**: defined in current shell or script
- **Environment variables**: exported, available to child processes
- **Shell variables**: predefined by the shell (`HOME`, `USER`, `PWD`)

### Exporting Variables
```bash
export PATH=$PATH:/usr/local/bin
```
- `export` makes a variable available to child processes

---

## Parameter Expansion
- Allows modifying variable values or substituting defaults

### Default Value
```bash
echo ${username:-guest}  # Prints 'guest' if username is unset
```

### Assign Default Value
```bash
echo ${username:=guest}   # Sets username to 'guest' if unset
```

### Length of Variable
```bash
echo ${#name}             # Prints length of 'Pujan' → 5
```

### Substring Extraction
```bash
echo ${name:0:3}          # Prints first 3 characters → Puj
```

### Replace Substring
```bash
echo ${name/L/LL}          # Replace first occurrence of 'L' with 'LL'
```

---

## Practical Example
```bash
#!/bin/bash
# Variables and default values
username=${USER:-guest}
echo "Hello, $username!"
file="/tmp/data.txt"
lines=${file:-0}
echo "Processing file $file, line count: $lines"
```

---

## Summary
- Variables store and reuse data
- `$variable` or `${variable}` to access value
- `export` to share with child processes
- Parameter expansion allows **default values, substring extraction, replacement, and length**
- Proper variable use makes scripts more **dynamic and flexible**
