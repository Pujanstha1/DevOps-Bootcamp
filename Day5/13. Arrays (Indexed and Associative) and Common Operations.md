# 13. Arrays (Indexed and Associative) and Common Operations

Arrays allow you to store **multiple values in a single variable**.

---

## 13.1 Indexed Arrays
- Elements accessed by numeric index starting from 0

### Defining an Indexed Array
```bash
fruits=(Apple Banana Cherry)
```
- Alternative:
```bash
fruits[0]=Apple
fruits[1]=Banana
fruits[2]=Cherry
```

### Accessing Elements
```bash
echo ${fruits[0]}   # Apple
echo ${fruits[1]}   # Banana
```

### All Elements
```bash
echo ${fruits[@]}   # Apple Banana Cherry
echo ${#fruits[@]}  # Number of elements → 3
```

### Loop Through Indexed Array
```bash
for fruit in "${fruits[@]}"; do
    echo $fruit
done
```

---

## 13.2 Associative Arrays (Bash 4+)
- Elements accessed by string keys

### Declaring an Associative Array
```bash
declare -A colors
colors[apple]=red
colors[banana]=yellow
colors[cherry]=dark_red
```

### Accessing Elements
```bash
echo ${colors[apple]}   # red
```

### All Keys and Values
```bash
echo ${!colors[@]}   # Keys: apple banana cherry
echo ${colors[@]}    # Values: red yellow dark_red
```

### Loop Through Associative Array
```bash
for key in "${!colors[@]}"; do
    echo "$key is ${colors[$key]}"
done
```

---

## 13.3 Common Operations
1. Adding elements:
```bash
fruits+=(Mango)
```
2. Removing elements:
```bash
unset fruits[1]  # Removes Banana
```
3. Array length:
```bash
echo ${#fruits[@]}  # Number of elements
```
4. Slicing array:
```bash
echo ${fruits[@]:1:2}  # Elements from index 1, length 2
```

---

## Practical Examples
1. List files in array:
```bash
files=(/etc/*.conf)
for file in "${files[@]}"; do
    echo $file
done
```
2. Map users to roles:
```bash
declare -A roles
roles[alice]=admin
roles[bob]=user
for user in "${!roles[@]}"; do
    echo "$user has role ${roles[$user]}"
done
```
